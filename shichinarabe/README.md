# 七並べ対決 記録アプリ

親戚での七並べ対決の「勝敗・順位・ドボン・賞金」を記録し、メンバーごとの収支を自動集計する単一ページアプリ。
GitHub Pages: https://ko-yamatoya.github.io/Projects/shichinarabe/

## 使い方
1. **名簿**：初回に11人（人数は増減可）の名前を入力（この端末のブラウザ内だけに保存。GitHubには送られません）。
2. **＋新しいチーム戦**：1回戦を追加。
3. **チーム編成**：各チームをペア/ソロで組み、A/Bグループへ割り当て（人数が減ったらチーム削除・全ソロも可）。
4. **A・Bグループ**：上位から順にタップして順位入力。ドボンした人はチェック＋ジョーカー枚数を選択。
5. A/Bの結果から **Winners / Losers** の顔ぶれが自動確定。そこでも順位・ドボンを入力。
6. **収支**が自動計算され、最下部の **総合ランキング** に全員の累計が反映される。

## ルール（計算仕様）
- 1チーム戦 = A / B / Winners / Losers の4試合。
- A上位・B上位が Winners、残りが Losers（進出人数は可変。既定 A3人・B3人）。
- 順位賞金は **Winners / Losers のみ** 発生（既定）:
  - Winners: トップから +500 / +400 / +300 / +200 / +100 / ±0（トップ基準・固定）
  - Losers: ボトムから −500 / −400 / −300 / −200 / −100（ボトム基準・固定）
  - 人数が減っても「トップ +500」「ボトム −500」は保たれる。
- **ペアは順位賞金の合計を折半**。ソロは自分の賞金そのまま。
- **ドボン**（ペア無関係の個人清算・可変）:
  - パスドボン = 基本額（既定200円）
  - ジョーカードボン = 基本額 + ジョーカー1枚あたり加算（既定200円）× 枚数
  - ドボンした人が支払い、その試合の **1位が総取り**。
  - 複数ドボンは「先にドボンした人＝最下位」になるよう、下位から順にタップして入力。

## ファイル
- `index.html` … 画面・スタイル
- `calc.js` … 収支計算ロジック（正本）
- `app.js` … UI・状態管理（localStorage保存 / JSON書き出し・読み込み）
- `test.js` … 計算ロジックのテスト（`jsc -e "load('calc.js');load('test.js');"`）
- `smoke.js` … DOMスタブでの結合スモークテスト

データはブラウザのlocalStorageに保存。端末をまたぐときは「データ」→書き出し/読み込み（JSON）。
